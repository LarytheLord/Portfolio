The Problem

Communities worldwide lose battles against harmful development proposals — factory farms, polluting facilities, questionable zoning changes — because they lack the legal expertise to file proper objections within tight regulatory deadlines. A single factory farm permit can affect thousands of people, but the window to object might be 21 days and the legal language required is impenetrable.

The legal advocacy pipeline is broken: hiring lawyers costs thousands, deadlines are unforgiving, citation requirements vary by jurisdiction, and NGOs can't cover every permit in every country.

Origin: Code4Compassion Mumbai

Built in a 10-hour hackathon sprint. The original prototype — called AFOG (Automated Factory Farm Objection Generator) — could ingest a planning permit, analyze it using Google Gemini, and generate a legally-sound objection letter with relevant citations in seconds. Work that would normally take lawyers weeks.

We won the hackathon. But the real question was: could this scale beyond factory farms, beyond India, beyond a demo?

Scaling Through AARC Pre-Accelerator

Open Permit was accepted into the AARC Pre-Accelerator in Bangalore. The transformation was massive:

Rebrand: AFOG → Open Permit. The tech was jurisdiction-agnostic — it could handle any government permit, not just factory farms. The name needed to reflect that broader civic intelligence vision.

Multi-country expansion: Went from India-only to 8+ countries. Each country required understanding local permit filing systems, building legal framework parsers, adapting letter templates to local standards, and localizing AI prompts.

Fundraising infrastructure: Built a funder outreach database of 197 contacts across 75 countries. Multiple pitch deck iterations for AARC Demo Day (delivered March 7, 2025). Warm leads at Dasra. SFF grant pitch framing Open Permit as an institutional learning mechanism. Bulk outreach campaigns via Brevo targeting ~150 NGOs/funders across 30+ countries.

Technical Architecture

AI & NLP

Google Gemini API for permit analysis, legal text generation, and multi-language NLP processing across jurisdictions.

Backend

Express.js API managing a legal citation library, permit tracking system, and recipient routing for objection letters.

Frontend

Next.js interface with permit ingestion pipeline, chatbot infrastructure, and real-time status tracking.

Data Layer

Supabase with PostgreSQL. Permit data, legal frameworks, user submissions, and CSV-based bulk operations for outreach campaigns.

Full stack is open source under Apache 2.0, deployed on Vercel. The automation pipeline detects new permit filings and generates timely responses.

How Claude Code Manages This

Claude Code is deeply embedded in Open Permit's development. The entire Next.js + Express stack was iterated through Claude Code sessions — complex features like the permit ingestion pipeline, Gemini API integration, and recipient routing system all built through collaborative conversations.

Beyond code: every version of the AARC Demo Day pitch deck, the funder outreach database structure, the SFF grant framing, CSV data pipeline work (permit normalization, funder contacts, recipient routing, Brevo campaign structuring) — all developed through Claude Code.

Each new country's legal framework expansion — researching jurisdiction requirements, drafting legal templates, validating citation formats — runs through Claude Code sessions before going into the platform.
