Pattern 1: Architecture Sessions

When starting a new feature, I open a Claude Code session with full context — what exists, what needs to exist, what could go wrong. The back-and-forth surfaces edge cases, suggests patterns from other domains, and challenges assumptions.

Example: Knight Medicare's abstraction layer. When the Supabase India block hit, I needed to decouple the entire data layer from Supabase-specific APIs. Claude Code designed the abstraction so swapping providers requires changing adapter code, not business logic. That pattern now protects all 11 projects.

Pattern 3: Pitch & Strategy Work

A huge amount of Claude Code usage is non-technical. AARC Demo Day pitch deck iterations. SFF grant pitch framing Open Permit as an institutional learning mechanism. Dasra outreach strategy. ALPN collaboration drafts. Adventurers Guild financial model with Indian market rates. Every strategic document runs through Claude Code.

Pattern 4: OSS Contribution Workflow

For contributing to repos with thousands of stars: (1) codebase understanding — "explain how this module works," (2) PR preparation — clear descriptions matching repo conventions, (3) review iteration — when maintainers request changes, Claude Code helps iterate fast. This workflow produced 56+ merged PRs in 2026.

Pattern 5: Infrastructure Decisions

The Yotta evaluation (29 services, 11 projects, 63% cost reduction), SMS provider selection (MSG91 + Plivo dual approach), Supabase alternatives analysis — all complex tradeoff evaluations managed through Claude Code. Migration scripts, Docker Compose configs, health checks, rollback procedures — all built collaboratively.

What Claude Code Doesn't Do

Clinical decisions for Knight Medicare — that's the Harvard PhD's domain. Relationship building — funder meetings, hackathon networking, team management. Creative vision — "what should we build" decisions. Shipping discipline — deciding what to ship today vs. next month. Accountability — when code breaks at 2 AM, that's on me.

The Force Multiplier

Without Claude Code, I'd be running 2-3 projects. With it, I run 11. That's a ~4-5x force multiplier (conservative). The real value isn't any individual code generation — it's the compounding context across 100+ conversations about how all 11 projects relate.

The analogy: Batman doesn't fight crime because he has a Batcave full of gadgets. He fights crime because he has the judgment to use them effectively. The gadgets extend his reach. Claude Code is my Batcave.
